## Introduction
#### Why Learn Data Structures & Algorithms?

In a technical interview, it's not your knowledge of specific data structures or algorithms that's tested. It's your approach towards a problem.

**Solving problems demonstrates:**
- You can think about a problem **systematically** and solve it systematically step-by-step
- You can envision different inputs, outputs, and edge cases for programs you write
- You can communicate your ideas clearly to co-workers and incorporate their suggestions
- You can convert your thoughts and ideas into **working code** that's also **readable**

#### The Method

1. State the **problem** clearly
	- Identify input and output formats
	- Write the problem in your own words, make it as short as possible
2. Come up with some example **inputs** and **outputs**
	- Try to cover all edge cases
	- Represent test cases as dictionaries
3. Come up with a **correct solution** for the problem
	- State it in plain English
	- Doesn't have to be the most efficient solution yet
	- First aim for correctness, then efficiency
	- Come up with an algorithm 
4. Implement the **solution and test it** using example inputs
	- Fix bugs, if any
5. Analyze the algorithm's complexity and identify **inefficiencies**, if any
6. Apply the **right technique** to overcome the inefficiency
	- Repeat steps 3-6

## Error Handling
#### Debugging Tips

- Add clear print statements to see the output of the code
- Always implement a brute force solution
