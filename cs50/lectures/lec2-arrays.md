## Debugging

The process of finding issues (bugs) in code and fixing them.
#### Debugger

A piece of software that allows for debugging code by slowing down or pausing execution and walk through execution of the code line by line.
#### Its Components

- **Step Over** → executes the current line of code and moves to the next line in the _same_ function its currently looking at
- **Step Into** → tells the debugger to follow the execution path _inside_ any function call on the current line

## Compiling

**Compiling code refers to 4 main processes:**
1. Preprocessing
2. Compiling
3. Assembling
4. Linking
#### 1. Preprocessing

The compiler finds on the server's hard drive a file with the imported library's name, goes inside, and copies and pastes the relevant lines of code into our own code.
#### 2. Compiling

Taking the preprocessed code and convert it into Assembly code.
#### 3. Assembling

Converting the Assembly code into binary code (0s and 1s). 
#### 4. Linking

Link all the 0s and 1s together. It's the process of linking all the chunks of binary code from you own code and the imported libraries together into one executable file.
